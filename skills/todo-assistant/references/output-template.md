# 출력 마크다운 템플릿

저장 경로: `./todos/YYYY-MM-DD.md`

```markdown
# Todo — 2026-05-10 (일)

> **이번 주 목표**: 신규 기능 v1 배포
> **진행률**: 주간 3/7 완료

## 🌞 오늘 할 일 (Daily, 우선순위 순)

### [ ] D1. `[P0]` 설정 파일 항목 보강
- **시작점**: `configs/example.yaml` 열기
- **참조**: 이전 PR에서 사용한 패턴 문서
- **완료 기준**: 정의된 항목 25개 이상, 카테고리 5개 이상
- **막힐 때**: 항목이 떠오르지 않으면 brainstorming 도구 활용
- **예상 시간**: 60분
- 📎 근거: doc1.md L42-50

### [ ] D2. `[P1]` (depends_on: D1) 검증 스크립트 실행
- ...

## 📅 이번 주 (Weekly, 우선순위 순, ~ 2026-05-16)

- [x] W1. `[P0]` 데이터 스키마 확정 *(2026-05-08 완료)*
- [ ] W2. `[P0]` 설정 파일 확장 — **due: 2026-05-11**
- [ ] W3. `[P1]` 평가 파이프라인 구성 — due: 2026-05-13
- [ ] W4. `[P1]` v1 배포 — due: 2026-05-16

## 🔁 이전 미완료 이월

- D3 (2026-05-08): "외부 endpoint 인증 디버깅" → 오늘 D4로 이동

## 🧠 분석 메모

주어진 자료에서 핵심은 **외부 연동 설정 검증**. 인증 이슈가 반복되면
권한 설정을 먼저 확인할 것.

## 🔗 등록 결과

- Notion: https://notion.so/...   (3개 페이지 생성)
- Calendar: 2개 이벤트 생성 (D1: 09:00~10:00, D2: 10:30~12:00)
```

## 작성 규칙

- D1은 **항상 P0 + "시작점" 정보 명시**(첫 명령어/파일/링크) — 시동 부담 최소화
- 우선순위 태그(`[P0]`/`[P1]`/`[P2]`)는 항상 백틱으로 감싼다
- 완료된 weekly 항목은 `[x]` + `*(YYYY-MM-DD 완료)*` 표기
- 각 todo의 sources는 본문 끝에 `📎 근거:` 한 줄로
- `## 🔗 등록 결과` 섹션은 외부 등록을 수행했을 때만 추가
